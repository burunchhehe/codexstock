# Android Mobile Console

코덱스스톡 모바일 콘솔은 집 PC에서 실행되는 코덱스스톡 본체를 개인 안드로이드 기기에서 확인하기 위한 비공개 운영 화면입니다. 모바일 앱에 증권사 키나 계좌 원장을 복제하지 않고, PC 본체가 최종 데이터 소스가 됩니다.

## 구조

```text
Android 앱
    -> 개인 HTTPS 사설망
    -> PC 코덱스스톡 모바일 API
    -> 상태·후보·직원·하위엔진·사건 원장
```

- 화면은 경량 웹 콘솔과 Capacitor 안드로이드 셸로 구성됩니다.
- 처음 연결할 때 PC 주소, 짧은 일회용 코드, 기기 이름을 사용합니다.
- 일회용 코드는 만료되고 재사용할 수 없습니다.
- 장기 기기 토큰은 원문이 아니라 SHA-256 해시만 PC에 저장합니다.
- 연결된 기기는 PC에서 개별 폐기할 수 있습니다.
- 공개 저장소에는 실제 PC 주소, 연결 토큰, 기기 식별값을 포함하지 않습니다.

## 모바일에서 확인할 수 있는 것

- 본체 연결과 마지막 응답 지연
- 현재 운영 상태와 장중·장후 업무 집중도
- 자동운용 모드와 외부 실행기 상태
- 추천·관찰 후보와 핵심 근거
- AI 직원 및 하위엔진 상태
- 내부 개발자 사건과 복구 상태
- 계좌 요약의 공개 가능한 범위
- 읽기 전용 비서 질의
- 긴급정지 요청

## 의도적으로 막은 기능

모바일 콘솔은 편의를 위해 실전 안전 경계를 낮추지 않습니다.

- 종목 직접 매수·매도
- 자동매매 시작 또는 잠금 해제
- 위임 비중과 손실 한도 완화
- API 키·계좌번호·주문 토큰 열람
- 운영 원장 직접 수정
- 소스코드 수정

긴급정지는 위험을 줄이는 단방향 안전 동작이므로 별도 확인창을 거쳐 허용합니다. 다시 시작하거나 제한을 완화하는 동작은 PC의 기존 안전 절차를 따라야 합니다.

## 연결 방식

실제 검증에서는 Tailscale의 개인 tailnet과 HTTPS Serve를 사용했습니다. Tailscale은 구현의 필수 종속성이 아니라 현재 선택한 사설망 전송 수단입니다. 같은 안전 조건을 충족하는 개인 VPN이나 고정 HTTPS 역방향 프록시로 교체할 수 있습니다.

공개 저장소의 예시에는 개인 tailnet 주소를 넣지 않습니다.

## 검증 결과

- 안드로이드 실제 기기 연결과 일회용 페어링 성공
- 기기 토큰 해시 저장과 폐기 검증
- 모바일 390x844 화면에서 가로 넘침과 버튼 잘림 없음
- 후보 새로고침과 읽기 전용 비서 응답 확인
- 매수·자동매매 시작·위험 완화 명령 차단 확인
- 긴급정지 확인창 동작 확인
- Android unit test와 lint 통과
- `npm audit` 취약점 0건
- 디버그 APK v2 서명 확인

공개 저장소에는 재현 가능한 소스와 빌드 스크립트만 포함하며, 로컬에서 만든 디버그 APK와 서명키는 포함하지 않습니다.

관련 구현은 [`app/mobile_console.py`](../app/mobile_console.py), [`app/web/mobile/`](../app/web/mobile/), [`mobile/codexstock-android/`](../mobile/codexstock-android/)에서 확인할 수 있습니다.
