## Current conclusion

The remediation MVP is functional, but the complete original specification is not finished. The honest overall completion estimate remains 99.9%. The integrated readiness gate now reports `engine_operational_ready=true`, `research_evidence_ready=true`, and `full_spec_evidence_ready=false`; it never enables automatic live trading. The persisted research tier is backed by `exp_97779bdfe3b046e98b15992e96ba47bd`: 2,444 source-pinned adjusted OHLCV rows, realistic delayed fills, passing point-in-time evidence for the requested exact listing interval, three strict OOS folds, 25 local robustness candidates and a verified JSON/Markdown/XLSX/CSV bundle. Its 52.36% maximum drawdown makes it evidence of the pipeline, not a strategy recommendation. Regime evidence shows losses of 87.37% when compounding BEAR bars and 33.73% across SIDEWAYS bars. Benchmark attribution further shows that the 3,392.72% strategy return lagged the underlying's 8,585.90% buy-and-hold return by 59.79% geometrically, with a -0.523 information ratio. The lifecycle now blocks this experiment from paper nomination on four independent grounds: negative geometric excess return, negative information ratio, drawdown beyond -50%, and catastrophic regime loss. A real read-only KIND fetch verified a 6,463-byte HTML document as `sha256:08816199042b3e28fef1c64b989da679560449e92bfb6c25a97b648c28447c62`; it was deliberately not registered because document semantics were not automatically parsed. The remaining full-spec evidence gaps are exact historical listing dates for the 542 restored delisted intervals, verified HTS profiles, and completion of the full-session WebSocket soak. Readiness now exposes fresh active-run progress separately from immutable completed-run evidence. The current runtime doctor passes, all 57 automated tests pass, and the analytical store contains 1,084,818 rows across 532 symbols. Full historical membership, HTS compatibility and all-day losslessness remain explicitly unverified rather than being inferred.

Production KIS run `realtime_run_67e5c7ae52cf4befb4eee3033591706a` completed from 2026-07-13 10:41:50 through 15:35:02 KST in read-only mode with orders disabled. It captured 140,192 total messages, 140,070 data messages, 429,655 accepted events, 164 deduplicated retransmissions, 118 heartbeats and one reconnect. The final file hash verifies, but `in_session_quality_ok=false` remains authoritative, so readiness correctly retains `full_session_realtime_soak` as a blocker rather than overriding immutable evidence after the fact.

The current machine-readable performance bundle is self-hashed as `sha256:3655d5a2dae26a1cf3c8693e7d02ed60ae0096cacf170b91d34399fe74b9c1b1`. Readiness recomputes this whole-file hash before trusting its `passed` flag; a tampered timestamp or result therefore fails the gate.

The latest 20-iteration concurrency bundle is likewise self-hashed as `sha256:d3f17711d1c60f17fa39ec6174cf216a60f9f3ec0513ebfac62c56df42c75b2c`; readiness independently verifies it before accepting the concurrency gate.

The readiness API now exposes cumulative top-level blocker lists for operational, research, and full-spec tiers in addition to category-local checks. The live audit reports no operational or research blockers and exactly four full-spec blockers: `complete_historical_universe`, `verified_corporate_action_history`, `all_hts_profiles_verified`, and `full_session_realtime_soak`. The corporate-action adjustment and immutable registry machinery are implemented, but zero complete source-document-verified event histories are currently registered; implementation alone is not treated as production evidence.

Realtime run verification now validates evidence structure after recomputing the whole payload hash: schema/status, exact KIS read-only provider and safety flags, non-empty symbols, run/file identity, chronological timestamps, non-negative run and stream counters, unique KIS stream IDs, valid stream timestamps/event hashes, and the in-session quality type are mandatory. A structurally empty payload or negative stream counter with a valid recomputed hash is rejected and excluded from `run_count`; the existing real run passes the strengthened verifier. Failed/cancelled runs remain auditable, but research readiness now requires at least one completed run with real data messages and passing in-session quality rather than merely a nonzero run count. The full suite contains 61 passing tests.

Verified realtime-history responses explicitly label run gap fields as current-run-only and stream summaries as cumulative store state at finish. They also expose status counts and completed-data-run counts. Research readiness requires a completed run with data messages, accepted events, a non-empty stream summary, and passing run-scoped in-session quality. Duplicate retransmissions may legitimately exceed accepted stream events and remain explicit evidence rather than being misclassified as corruption.

Run evidence additionally requires an exact 32-hex run identifier matching its filename, timezone-aware chronological run/stream timestamps, and each stream must be a KIS tick/order-book lane whose symbol belongs to the run subscription. Self-hashed evidence with a naive timestamp or a stream linked to an unsubscribed symbol is rejected. The real persisted evidence and all 61 tests pass these gates.

For new run evidence, monotonic `elapsed_seconds` must agree with signed wall-clock start/finish timestamps within a bounded drift tolerance. Requested duration must be non-negative, duration flags must be booleans, termination reasons are enumerated, and `duration_reached` must be supported by the requested duration, elapsed duration, and completion flag. A payload rehashed after replacing a short run's elapsed time with 14,400 seconds is rejected.

Realtime evidence numeric fields are type-strict: run/stream counters must be JSON integers rather than coercible strings or booleans, durations must be finite JSON numbers, and NaN/infinity are rejected. A self-hashed payload with a string-form message count fails verification; the persisted real evidence and all 61 tests continue to pass.

The cancellation paths passed 100 consecutive targeted executions; a 300-write concurrent status-poll stress test passed 20 consecutive runs. The full suite now contains 60 passing tests.

No replay metadata, frame, index or timeline hash errors were found; the largest schema-v1 verification took 1.009s and schema-v2 full verification took 1.660s in this audit.
