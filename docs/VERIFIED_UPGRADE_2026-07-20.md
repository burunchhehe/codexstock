# CodexStock Verified Upgrade - 2026-07-20

## 목적

이번 업데이트의 목적은 AI 판단과 증권사 주문 집행을 분리하고, 외부 엔진의 설치 여부가 아니라 실제 왕복 실행 증거를 확인하는 것입니다.

## 1. 외부 자동집행기

코덱스스톡은 주문 의도를 만들고, 별도의 규칙 기반 실행기가 이를 독립적으로 재검사하는 구조를 사용합니다.

```text
후보 탐색과 근거 수집
    -> 위험 검사와 주문 의도 생성
    -> 서명된 실행 신호
    -> 외부 실행기의 독립 재검사
    -> Shadow/Paper 결과 원장
    -> 코덱스스톡 상태판과 보고 체계로 반환
```

실행기 관측판에는 다음 항목을 표시합니다.

- 프로세스 연결 및 하트비트 상태
- 현재 실행 모드
- 처리 완료 및 대기 신호 수
- 장시간 Shadow 관찰 진행률
- 후보 증거 축적 진행률
- 실전 주문 지원 및 차단 상태

현재 검증 모드는 Shadow이며 실전 주문은 차단되어 있습니다. 이 경계는 UI 문구가 아니라 실행기 상태 계약의 일부입니다.

## 2. NautilusTrader 격리 런타임

Windows 11의 코드 무결성 정책이 서명되지 않은 네이티브 Python 모듈을 차단하는 환경을 확인했습니다. 보안 정책을 완화하는 대신 WSL2 Ubuntu에 별도 Python 런타임을 구성하고 NautilusTrader를 그 안에서 실행하도록 어댑터를 보완했습니다.

런타임 진단은 이제 단순 패키지 버전 조회가 아니라 실제 BacktestEngine 및 Equity 객체 import까지 확인합니다. 선택된 실행 백엔드는 `wsl2_linux`로 기록됩니다.

검증된 항목:

- 실제 엔진 프로세스 왕복 실행
- 부분 체결 시나리오
- 의도적 미체결 시나리오
- 10초 주문 삽입 지연 시나리오
- 호가장 기반 실행 증거
- 결과 JSON 및 프로세스 반환 코드 보존

지연 검증은 오래된 지정가가 시장성 주문으로 바뀌는 우연에 의존하지 않도록 시장가 주문 시나리오로 분리했습니다.

외부 엔진 진단은 다음 정보를 잃지 않도록 개선했습니다.

- timeout 여부와 실행 시간
- 프로세스 반환 코드
- 표준 오류와 상세 오류

따라서 설치 파일이 존재한다는 이유만으로 연결 성공으로 표시하지 않고, 실제 작업 요청과 결과 반환이 확인돼야 왕복 검증 상태가 됩니다.
